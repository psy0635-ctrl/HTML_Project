# 🕶️ SY_ERR  
### _Digital Fashion Editorial & Archive_

> **"The noise is the canvas."**

---

## 📌 1. 프로젝트 개요 (Overview)

| 항목 | 내용 |
|------|------|
| 프로젝트명 | SY_ERR (System Error) |
| 프로젝트 유형 | 디지털 패션 아카이브 웹사이트 |
| 핵심 목적 | 패션 아이템을 데이터로 관리하고 향후 코디 추천 시스템으로 확장 |
| 대상 사용자 | 자신의 패션 스타일을 체계적으로 관리하고 싶은 사용자 |

---

## 🎯 2. 프로젝트 목표

> **“패션을 데이터로 저장하고, 그 안에서 코디를 추천하는 시스템”**

---

## 📊 3.  핵심 목표 구조

```
[ 패션 아이템 입력 ]
↓
[ 데이터화 ]
↓
[ 필터링 ]
↓
[ 조합 ]
↓
[ 코디 추천 결과 출력 ]

👉 단순 웹사이트가 아닌 데이터 기반 시스템 설계

```
---

## 🧠 3. 시스템 동작 흐름 (Process Flow)

### 📊 전체 흐름

| 단계 | 기능 | 설명 |
|------|------|------|
| 1단계 | 데이터 입력 | 패션 아이템 정보 저장 |
| 2단계 | 데이터 구조화 | 속성별로 정리 |
| 3단계 | 필터링 | 조건에 맞는 데이터 추출 |
| 4단계 | 조합 | 코디 생성 |
| 5단계 | 출력 | 사용자에게 추천 결과 제공 |

---

### 📦 1단계: 데이터 입력 (Input)

사용자의 패션 아이템을 다음과 같은 형태로 저장

| 속성 | 예시 |
|------|------|
| 카테고리 | 자켓, 바지, 신발 |
| 색상 | Black, White |
| 스타일 | Street, Casual |
| 계절 | Winter, Summer |

---

### 🔎 2단계: 필터링 (Processing)

사용자가 조건을 입력하면 해당 조건에 맞는 데이터만 추출


조건:

- 스타일 = Street <br>
계절 = Winter

👉 결과: 조건에 맞는 아이템 목록

---

### 🤖 3단계: 추천 (Output)

필터링된 데이터를 기반으로 조합 생성


추천 코디:

Black Jacket <br>
Denim Pants <br>
White Sneakers

👉 핵심: **사용자가 가진 데이터 안에서만 추천**

## 🚀 5. SPA 구조 (Single Page Application)

### 📌 구조 비교

| 구분 | 기존 웹 | SPA |
|------|--------|-----|
| 페이지 이동 | 새로고침 발생 | 없음 |
| 속도 | 느림 | 빠름 |
| 사용자 경험 | 단절됨 | 부드러움 |

---

### 📈 SPA 동작 흐름

```
사용자 클릭
↓
JavaScript 실행
↓
DOM 변경
↓
화면 업데이트 (새로고침 없음)
```
